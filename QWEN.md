# igw-gateway

## Project Overview

`igw-gateway` is a Spring Cloud Gateway service — a reactive API gateway for the ThinkHub platform. It is the entry point that routes external traffic to downstream ThinkHub microservices.

- **Group / organization:** `uz.thinkhub` (Uzbekistan-based "ThinkHub" platform)
- **Root project name:** `igw-gateway`
- **Java package:** `uz.thinkhub.igw_gateway`
  - Note: `HELP.md` warns that the originally requested package `uz.thinkhub.igw-gateway` is invalid (hyphens are illegal in Java package names); Spring Initializr renamed it to `uz.thinkhub.igw_gateway` with an underscore. Keep using the underscored form.
- **Single-module** Gradle project.

## Building and Running

Use the Gradle wrapper. No global Gradle install is required.

```bash
# Run the test suite
./gradlew test

# Build an executable jar (in build/libs/)
./gradlew build

# Run the gateway locally on the default port (8080)
./gradlew bootRun

# Package a container image (Spring Boot 4.x OCI support)
./gradlew bootBuildImage
```

The main class is `uz.thinkhub.igw_gateway.IgwGatewayApplication`.

## Development Conventions

- **Build file language:** Kotlin DSL (`build.gradle.kts`). Keep new build logic in Kotlin DSL — do not introduce a Groovy `build.gradle`.
- **Dependency versions:** Spring Boot and Spring Cloud versions are managed centrally (`springBootVersion` plugin, Spring Cloud BOM imported in `dependencyManagement`). Add new starters via `implementation(...)`; do not pin Spring Cloud artifacts directly.
- **Java toolchain:** Java 25. Do not downgrade — the toolchain block in `build.gradle.kts` enforces this.
- **Package layout:** All sources live under `uz.thinkhub.igw_gateway`. Sub-packages should be added under this root as the codebase grows.
- **Configuration:** Externalize settings to `src/main/resources/application.yaml` (or profile-specific `application-<profile>.yaml`). Avoid hard-coding URLs, ports, or secrets in Java code.
- **Tests:** Use JUnit 5 with Spring Boot's testing support. The existing `IgwGatewayApplicationTests` is a context-loads smoke test — keep it green; new tests should follow the same `*Tests` naming under the matching package.
- **Generated/IDE files:** `.idea/`, `*.iml`, `build/`, `.gradle/`, `.vscode/`, etc. are git-ignored. Do not commit them.
