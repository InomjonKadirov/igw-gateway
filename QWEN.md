# igw-gateway

## Project Overview

`igw-gateway` is a **Gradle monorepo** for the ThinkHub Integration Gateway (IGW) — a Spring Cloud Gateway service that replaces a legacy PHP/Yii2 monolith using strangler-fig migration. It is the entry point that routes external traffic to downstream ThinkHub microservices and ~25 external banking/PSP providers.

- **Group / organization:** `uz.thinkhub` (Uzbekistan-based "ThinkHub" platform)
- **Root project name:** `igw-gateway`
- **Java package:** `uz.thinkhub.igw_gateway` (in `igw-edge/`, package is `uz.thinkhub.igw.gateway`; the underscore in the original Spring Initializr package was renamed to a dot for the new structure)
- **Multi-module** Gradle build with monorepo layout.

The full architecture rules, gotchas, and the per-phase workflow live in `docs/dev-prompts/igw-claude-code-prompts.md`. The currently-approved implementation plan for Phase 0 (Foundation) is in this repo's chat history and in `docs/adr/0001..0004.md`.

### Tech stack

| Concern | Choice |
| --- | --- |
| Language | Java 25 (Gradle toolchain pinned via `JavaLanguageVersion.of(25)`) |
| Build tool | Gradle 9.5.1 (Kotlin DSL — `build.gradle.kts`) |
| Framework | Spring Boot **4.1.0** |
| Gateway | `org.springframework.cloud:spring-cloud-starter-gateway` (Spring Cloud 2025.1.2) — the server-webmvc variant; no WebFlux in our code |
| Tests | JUnit 5 (Jupiter) via `spring-boot-starter-test` + Testcontainers + WireMock + ArchUnit |
| Resilience | Resilience4j 2.2.0 (`io.github.resilience4j:resilience4j-spring-boot3`) + Apache HttpClient 5 |
| Rate limit | Bucket4j 8.10.1 (`com.bucket4j:bucket4j-core`) |
| JWT | Nimbus JOSE 9.40 (`com.nimbusds:nimbus-jose-jwt`) |
| Config | `application.yaml` (per subproject) |
| Modulith | Spring Modulith 2.1.0 — wired into the build but passes trivially in Phase 0 (real enforcement in Phase 1+ for `igw-hub`) |

## Building and Running

Use the Gradle wrapper. No global Gradle install is required.

```bash
# Run the test suite
./gradlew test

# Build an executable jar (in build/libs/) for every subproject
./gradlew build

# Run the gateway locally (PR #1: placeholder; PR #8+: real routing)
./gradlew :igw-edge:bootRun
# OR run the built jar directly:
java -jar igw-edge/build/libs/igw-edge-0.0.1-SNAPSHOT.jar

# List all subprojects
./gradlew projects

# Package a container image (Spring Boot 4.x OCI support)
./gradlew :igw-edge:bootBuildImage
```

The main class is `uz.thinkhub.igw.gateway.IgwEdgeApplication` (for `igw-edge`) and `uz.thinkhub.igw.echo.EchoServerApplication` (for `local/echo-server`).

## Development Conventions

- **Build file language:** Kotlin DSL (`build.gradle.kts`). Keep new build logic in Kotlin DSL — do not introduce a Groovy `build.gradle`.
- **Convention plugins:** shared build logic lives in `build-logic/conventions/` as precompiled script plugins. Subprojects apply them by id (`id("java-conventions")`, etc.). Don't duplicate conventions per-subproject.
- **Dependency versions:** Spring Boot and Spring Cloud versions are managed centrally via `gradle/libs.versions.toml` and the BOMs imported in `spring-boot-app-conventions` / `spring-boot-library-conventions`. Add new starters via `implementation(libs.findLibrary("...").get())`; do not pin Spring Boot / Spring Cloud artifacts directly.
- **Java toolchain:** Java 25. Do not downgrade — the toolchain block in every convention plugin enforces this.
- **Package layout:** Each subproject has its own package under `uz.thinkhub.igw.*`:
  - `igw-edge` → `uz.thinkhub.igw.gateway`
  - `local/echo-server` → `uz.thinkhub.igw.echo`
  - The 6 starters → `uz.thinkhub.igw.starter.<feature>` (envelope, audit, error-mapping, resilience, token-cache, i18n)
  - `golden-tests` → `uz.thinkhub.igw.golden`
- **Configuration:** Externalize settings to `src/main/resources/application.yaml` (or profile-specific `application-<profile>.yaml`) in the consuming subproject. Avoid hard-coding URLs, ports, or secrets in Java code.
- **Tests:** Use JUnit 5 with Spring Boot's testing support. Starter tests use `ApplicationContextRunner` (or its web variants). App tests use `MockMvc` and `@SpringBootTest`. Architecture rules are enforced with ArchUnit.
- **Generated/IDE files:** `.idea/`, `*.iml`, `build/`, `.gradle/`, `.vscode/`, etc. are git-ignored. Do not commit them.
- **Money operations:** NEVER auto-retry a timed-out POST. Status-check-then-retry via the provider's check/status method. (This is enforced in the per-adapter modules in Phase 1+; the resilience starter's HTTP client has retry disabled by default.)
- **Audit:** Phase 0 ships the `audit-spring-boot-starter` skeleton; PR #7 implements the fail-closed interceptor and the `request` table. Raw card numbers must never reach the database.
